2. footer.php INLINE <style> BLOCK CONFLICTS WITH AND OVERRIDES style.css
   Files: footer.php, style.css

   footer.php contains a full <style> block that redefines rules already
   declared in style.css. Because <style> blocks in the <body> are parsed
   after the external <link rel="stylesheet"> in <head>, the inline block
   always wins at equal specificity.

   Conflicting properties:
     .footer-grid  grid-template-columns:  style.css = "1fr 1fr"
                                           footer.php = "auto 1fr"  <-- WINS
     .footer-grid  gap:                    style.css = "40px"
                                           footer.php = "30px"      <-- WINS
     .footer-col   font-size:              style.css = "0.95rem"
                                           footer.php = "0.9rem"    <-- WINS

   Additionally, style.css has a bare "footer { ... }" element selector,
   but footer.php uses <footer class="site-footer"> styled by .site-footer.
   These are split definitions that are easy to lose track of.

   Recommendation: Remove .footer-grid and .footer-col from style.css
   entirely since footer.php owns them, OR consolidate all footer rules
   into style.css and remove the <style> block from footer.php.

5. HARDCODED #ff6600 INSTEAD OF var(--flame) IN nav.php AND footer.php
   Files: nav.php, footer.php

   Both inline <style> blocks hardcode the brand orange (#ff6600) instead
   of using the CSS variable var(--flame). Inline <style> blocks CAN use
   CSS variables from :root — the browser resolves them at render time
   regardless of where the <style> tag sits.

   Affected properties in nav.php:
     border-bottom: 2px solid #ff6600
     .nav-links a.active-page { color: #ff6600; border-bottom: 2px solid #ff6600; }

   Affected properties in footer.php:
     border-top: 6px solid #ff6600
     .footer-col a { color: #ff6600; }

   Recommendation: Replace all #ff6600 instances with var(--flame) so a
   single color change in :root updates everything.

10. content: "" APPLIED TO REAL HTML ELEMENTS (INVALID CSS)
    File: style.css

    .global-break and .mobile-break both use:

    The "content" property only works on CSS pseudo-elements (::before,
    ::after). Applying it to real HTML elements has no effect and is invalid.
    The display: block and margin-top properties are doing all the work.

    Recommendation: Remove the "content: "";" lines from .global-break
    and .mobile-break.
